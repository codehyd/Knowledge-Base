# 02 · 垂直与套餐

## 1. 垂直（内容场景，二选一钉 30 天）

Docker 是 **交付形态**；里面装的仍是垂直知识，避免卖成空洞「帮装 Docker」。

| 代号 | 垂直 | 谁付钱 | 包内预置什么 | 验收 |
|------|------|--------|--------------|------|
| **A** | 店铺客服话术 | 店主 | 按其资料整理的话术库 + 拒答 | 能答政策题 + 拒答库外 |
| **B** | 课程答疑 | 老师/机构 | 讲义要点库 + 拒答 | 能答课内题 + 拒答超纲 |

也可卖「空模板 Docker」（仅结构+拒答预设），但 **主推带垂直资料的整包** 更好成交。

## 2. 套餐（按 Docker 交付设计）

价格自定：`预估小时 × 时薪 × 1.5～2`。

### 体验档

- 通用/小样 Compose 模板 + 短文「如何运行」  
- 不含深度整理资料，或不含远程协助  
- **交付**：压缩包（compose、说明）+ 文字指导  

### 标准档（主推）

- 按客户垂直整理知识（分类 1..N、摘要视情况）  
- 拒答提示词配好  
- **完整 Docker 交付物**（见 [06](./06-工具与成本.md)）  
- 远程/文档协助 **跑通一次**（客户本机或客户自己的服务器）  
- 验收题通过；资料导出 ZIP  
- 含有限次修改（如 2 次）  
- **交付**：Docker 包 + 运行文档 + 备份 ZIP + 验收记录  

### 管家档（选配）

- 标准档 + 包月：补资料、帮改 compose/小问题 N 次  
- **不含**：无限代运维、你出云主机钱  

### 加价：部署到客户云

- 客户已有轻量云/NAS，你 SSH 帮他装好并跑通 → 单独加价  
- 仍是 **跑在客户资源上**，不是你的云  

## 3. 商品说明必写条款

1. 交付为软件包与配置，默认 **客户自备运行环境**（Docker 可用）。  
2. 运行产生的模型费用由客户 API Key 承担。  
3. 只按导入资料作答；无资料则拒答。  
4. 素材合法由客户保证。  
5. 协助上线次数/时长写死；不含你家 NAS 长期托管。  
6. 含资料导出备份。  

## 4. 拒单

- 只要「你给我挂个网站一直能用」且不愿自备机器/云 → 现阶段可拒或明确加价「以后云托管」且先不做  
- 不会/不肯装 Docker 且拒绝任何有 Docker 的环境 → 体验档仅文档，或劝其先租带一键 Docker 的机器  
- 万能 AI、灰产、无资料空壳乱要价
