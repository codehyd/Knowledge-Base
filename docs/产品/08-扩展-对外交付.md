# 08 · 扩展：对外交付（可选 · 副业）

> 仅在 [07](./07-打磨计划.md) 自用 / 认知闭环稳定后再看。这里不是起步目标。

## 1. 若要对外，卖的是什么

| 内容 | 说明 |
|------|------|
| **交付物** | 可 `docker compose up` 的 **空库** 知识库包 + 运行文档 +（可选）喂养 / 归类说明 + 拒答配置 |
| **跑在哪** | 对方自己的电脑 / NAS / 云；不是默认挂在你家 |
| **Key** | 对方自备 API Key（如 DeepSeek） |
| **你可协助** | 远程协助跑通一次（可另计）；不承诺无限期免费代运维 |

对方付钱买的是：**能在他自己环境跑起来、自己喂养的空库能力**，不是预装百科，也不是长期挂在你 NAS 上的账号。

## 2. 边界

```
自家 NAS Docker     →  研发、自测、录演示（你自己的认知库）
        ↓
交付物：镜像/Compose/文档/空库能力
        ↓
对方机器 / 对方云   →  对方填 Key、自己喂养（你可协助一次）
```

## 3. 交付流水线（摘要）

```
确认有运行环境 → 说明空库原则与拒答 → 你在 NAS 打出 Docker 包
  → 对方侧运行并自备 Key（可协助）
  → 对方试喂一小份材料做验收 → 交文档与（可选）导出说明 → 复盘
```

交付包至少含：`docker-compose.yml`、`.env.example`、`README-运行.md`、验收题（能喂 / 能答 / 会拒）。  
**不要**打进真实 Key、你家生产密钥、侵权预置库。

## 4. 交付话术示例

> 这是一套默认为空的知识库：请按《运行说明》在你的环境启动，把 API Key 写入 `.env`。  
> 自行上传电子书 / 笔记等喂养；AI 会帮你归类归纳；对话只按库内答，超出范围会拒答。  
> 默认不提供长期云托管；若需帮你装到你自己的服务器，可另约。

## 5. 拒单（扩展期仍适用）

- 只要「你给我挂个网站一直能用」且不愿自备机器 → 可拒或明确「以后云托管」且先不做  
- 不肯装 Docker 且无任何可用环境 → 仅文档体验，或劝其先有一带 Docker 的机器  
- 要求预装通识百科 / 万能 AI、灰产、无资料却要「什么都能答」
