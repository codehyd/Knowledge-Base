# 11 · 附录：未来蓝图（非当前 KPI）

> 仅在通过 [10](./10-扩展-市场与闸门.md) 相关闸门后参阅。认知 / 自用期不要按本章排期。

## 1. 方向（过闸门后）

在自用稳定、（若做）交包也顺之后，才可选：

- 你方轻量云 **托管套餐**（另定价）  
- 或薄产品官网 + Freemium（仍默认空库、自备 Key）  

当前附录不指导开工顺序。

## 2. 能力分层（将来）

| 层 | 内容 |
|----|------|
| 喂养 | 电子书 / 笔记 / 链接；视频与网页可自动提取文案（字幕 / 转写） |
| 净化 | AI 分类 1..N、摘要、切分、向量化；支持人工改标签 |
| 出水 | 检索问答、拒答、引用出处 |
| 信任 | 导出 ZIP、自备 Key、空库原则 |
| 增长 | 迷你样例、分享访客页（访客不可导出全文） |

## 3. 技术轮廓（将来选型参考）

- 前端：Web / H5；小程序后置  
- 后端：任选熟悉的 Node 或 Python  
- 数据：Postgres + pgvector（或等价）起步  
- 模型：OpenAI 兼容，用户 Key 路由（DeepSeek 等）  
- 部署：以 **Compose 自托管** 为主；你方云托管仅闸门后另议；不上 K8s  
- 队列：大批量喂养异步处理（有量再上）

## 4. 数据对象（将来）

用户、AI 配置（Key / 模型）、知识库（可多库）、条目、分类关联、切片、导出任务、（可选）会话与邀请码。

## 5. 与自用的关系

自己的真实喂养与提问 = 最好的探针。  
产品只解决「重复劳动与标准化」，不改变 **「先提升认知，后可选副业」** 的顺序；也不改变 **空库自喂** 的原则。
