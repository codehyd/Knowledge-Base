# 02 · 喂养与归类

默认 **空库**：没有知识兜底。价值来自你投递什么、AI 怎么帮你整理。

## 1. 可投递的材料（喂养入口）

| 类型 | 例子 | 处理要点 |
|------|------|----------|
| 电子书 | PDF / EPUB / TXT 等 | 抽取正文 → 切片 → 摘要 / 分类 |
| 笔记 | Markdown、纯文本、手记导出 | 一条一事更好检索 |
| 视频链接 | 课程、演讲 URL | **自动提取文案**（字幕抓取 / 语音转写），无需用户手贴文稿；失败时再允许补贴 |
| 其它 | 网页剪藏、表格导出 | 合法、你有权使用 |

原则：**只进你有权使用的材料**；侵权 / 灰产不进库。

## 2. 喂养举例（端到端）

```
投递《脑筋急转弯》电子书
        ↓
抽取题目、解析、章节结构
        ↓
AI 归类：益智 / 心理学 / …（可多选）
        ↓
写入条目 + 摘要，便于日后对话检索
```

再喂一本中医入门书 → 另打「中医」「方剂」等分类；与益智库 **共存**，不互相覆盖。

## 3. 归类规则

- 每条：**至少 1 个分类**，可以多个；能多则多，窄则一个，禁止硬凑。  
- 分类名用你自己能搜到的词（「益智」「中医基础」），不为好看造空层级。  
- 摘要：有助于检索再写；没把握就保留原文切片。  
- AI 给出的分类应 **可人工改**：你才是最终认知主。

## 4. 入库流水线（自用）

```
收集材料 → 投递上传 → AI 抽取 / 切片 / 摘要 / 打分类
        → 你抽查改标签 → 对话自测（能答 / 拒答）→ 备份
```

| 步骤 | 要点 |
|------|------|
| 收集 | 合法、对自己有认知价值 |
| 投递 | 固定入口（上传 / 粘贴链接 / 导入目录） |
| AI 整理 | 依赖你配置的 API Key；费用按用量计 |
| 抽查 | 分类是否离谱、摘要是否失真 |
| 自测 | 见 [03](./03-拒答与质量.md) |
| 备份 | 见 [05](./05-导出与备份.md) |

## 5. 空模板 vs 已喂养的库

| 形态 | 用途 |
|------|------|
| 空库（结构 + 拒答预设 + Key 配置位） | 开箱即用的软件形态；**默认就是这样** |
| 已喂养的个人库 | 日常涨认知的真正资产 |

软件交付的是 **空库能力**；知识资产永远是你自己喂出来的。

## 6. 多领域怎么管

不必「先钉死一个垂直再做软件」。自用期建议：

1. **先喂一个你最近真会问的主题**（例如益智题），把链路跑通。  
2. 通了再加第二主题（例如中医），验证多分类 / 多库不打架。  
3. 对话时可按分类过滤，也可全库检索——产品上预留即可，自用期先保证「喂得进、找得到」。

## 7. 和「以后副业」的关系

若某天要把同一套能力交给别人：空库 + 喂养 + 归类 + 拒答仍是核心——那是自用沉淀。套餐与话术见 [09](./09-扩展-获客与套餐.md)。
