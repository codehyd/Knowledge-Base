# 06 · 自测与样板

## 1. 目标

证明：**空库能喂、能归类、Docker 下能跑、对话会拒答**。  
不依赖公网 24h 托管。

## 2. 推荐自测方式（由易到难）

1. **本机 / NAS**：`compose up` → 配 Key → 打开网页 → 喂一份小材料 → 能答 + 拒答各做一轮（见 [03](./03-拒答与质量.md)）  
2. **录屏 / 截图**：留给以后自己复盘，或偶尔给人看  
3. （可选）会议共享屏幕  
4. （可选）短期穿透公网——注意安全，用完关掉  

## 3. 建议用的自测材料

| 材料 | 验证点 |
|------|--------|
| 一小本益智 / 脑筋急转弯（或摘录几章） | 喂养 → 多分类 → 「好玩的题」类问题能答 |
| （第二批）中医或其它领域摘录 | 多领域共存、互不污染 |
| 故意问库外题 | 必须拒答 |

样板材料用 **脱敏、你有权使用** 的内容；与日常真库分开维护，避免演示数据污染认知库。

## 4. 样板包（软件侧）

做一个 **空库 + 可选迷你样例** 很有用：

- 验证 Compose / 文档是否写得够清楚  
- 换真资料前先验证「上传 → 归类 → 对话 → 拒答」链路  

默认仍是空库；样例可一键导入、也可不导入。

## 5. 自测通过后再谈副业

自用稳定、自己真在涨认知前，不必排获客与上架。若以后要对外，样板可顺手当演示素材（见 [08](./08-扩展-对外交付.md)）。
