# 07 · 打磨计划（认知优先）

## 1. 四周日历

| 周 | 目标 | 动作 |
|----|------|------|
| **W1** | 家里跑通 | NAS 上 Compose 起空库；配好自己的 API Key；能上传一小份材料并对话；写出《运行说明》草稿 |
| **W2** | 真喂养 | 喂进一个你真会问的主题（如益智题）；AI 归类 + 你抽查；做一次完整备份 |
| **W3** | 日常真用 | 每天真实提问；记下卡点；只改文档 / 脚本 / 喂养质量，不堆功能 |
| **W4** | 复盘 + 多领域试水 | 评估认知是否真涨；试喂第二领域（如中医摘录）；**暂不排副业** |

### 成功线（自用）

- 空库 → 喂养 → 归类 → 对话 → 拒答 闭环稳定  
- 自己愿意每周都打开用  
- 备份流程固定  

### 失败信号

- 功能越加越多，自己却不喂、不问  
- 资料一团糟仍想上云 / 对外卖  
- 靠通识胡答「感觉很聪明」，拒答形同虚设  

## 2. 时间盒

每周若干小时即可：打磨 Compose、喂养资料、自测对话。  
**不花时间**买云做托管平台、不对标通用产品功能表、不上架闲鱼。

## 3. 何时才看「扩展 / 副业」文档

满足大部分再打开 [08](./08-扩展-对外交付.md) 起：

- 自己用得顺，认知上有收获  
- 换一批资料 / 加一个领域不太痛  
- 运行说明自己三个月后还看得懂  

未满足：继续留在 01～07。

## 4. 禁止清单（认知 / 自用期）

- [ ] 默认用 NAS 给外人当生产  
- [ ] 未自用稳定就租云做平台  
- [ ] 空 Docker 不谈喂养与拒答  
- [ ] 包里 / 仓库里塞真实 API Key  
- [ ] 把「先成交」排在「自己能用、能涨认知」前面  
- [ ] 对标通用云产品功能军备  
- [ ] 预装通识百科当主卖点，放弃空库原则
