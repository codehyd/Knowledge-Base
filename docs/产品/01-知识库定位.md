# 01 · 知识库定位

## 1. 你现在在做什么

| 角色 | 内容 |
|------|------|
| **先为自己** | 做一款 **可自托管的个人知识库**：默认没有知识兜底，靠自己投递资料「喂养」；用 AI 总结、归类、归纳；再通过对话复习、检索、串联认知 |
| **交付形态（自用）** | **Electron + 本地 SQLite**：能喂养、能对话、会拒答、可备份；不依赖 Docker |
| **副业** | **以后再说**；先证明自己真的在用、真的涨认知 |

你要的是：**自己投递 → AI 整理 → 只按库内对话 → 认知可积累**；不是先做一个对外卖的商品，也不是通识聊天机器人。

## 2. 产品一句话

> **空库知识库**：没有预置百科。你喂什么，它懂什么；你没喂的，它拒答。  
> 支持多个领域并存（脑筋急转弯、中医、工作笔记……），各自归类、可交叉检索。  
> **大模型 API Key 由用户自备**（如 DeepSeek），费用与模型选择自己控。

## 3. 核心能力（MVP）

| 能力 | 说明 |
|------|------|
| **喂养** | 投递电子书、视频链接、笔记等认知材料 |
| **归类归纳** | AI 吸收后打标签 / 分类（可多分类），并做摘要便于检索 |
| **对话** | 基于已喂养内容问答、归纳、找例子 |
| **拒答** | 超出库内范围 → 明确拒绝，不靠通识瞎编 |
| **自备 Key** | 配置自己的模型 API Key 后才能对话 / 入库处理 |

### 喂养举例

上传一本《脑筋急转弯》电子书 → 系统吸收书中题目与解析 → 归类为「益智」「心理学」等一个或多个分类点。

### 对话举例

已喂养上述书后，问：「有哪些比较好玩的题目？」→ 按库内归纳作答。  
若问库中从未出现的内容（如某政策、某考试考点）→ 拒答。

### 多库 / 多领域

不是单一垂直。你还可以再喂中医书籍、工作流程笔记等；同一套软件承载 **多个主题的知识积累**。

## 4. 优先级（务必分清）

```
1. 认知闭环     →  能喂、能归类、能问、会拒答、自己愿意天天用
        ↓
2. 运行稳定     →  Electron / 本机可复现、备份可带走、文档自己看得懂
        ↓
3. （可选）副业 →  自用跑通后再谈交包 / 获客（见 08～10）
```

| | 做 | 不做（现阶段） |
|--|----|----------------|
| 空库 + 自喂 | 主路径 | 预装通识百科当卖点 |
| 多领域 | 允许并存、可分类 | 强行钉死「只能做一个垂直」 |
| Key | 用户自备（DeepSeek 等） | 包里塞你的 Key、代付 Token |
| 副业 | 文档里预留扩展 | 先排上架、先对标云平台 |

## 5. 和通用 AI 怎么相处

| | 通用聊天 | 你的知识库 |
|--|----------|------------|
| 知识来源 | 模型通识 + 联网 | **只有你喂进去的资料** |
| 价值 | 什么都能聊一点 | **边界清晰：懂你喂过的，拒你没喂的** |
| 数据 | 多在平台云上 | **资料在自己环境** |
| Key | 平台账号 | **你自己的 API Key** |

仍可基于开源组件自研组装；核心是 **你的资料与归类**，不是对标某大模型产品。

## 6. 对自己的核心承诺

| 承诺 | 自己感知 |
|------|----------|
| 空库可喂 | 投递后能被吸收、归类、检索 |
| 只答库内 + 会拒答 | 敢当真用，不怕乱编带偏认知 |
| 多领域 | 脑筋急转弯与中医可以同机共存 |
| Key 自备 | 模型费用与选型自己控 |
| 资料可导出 | 不绑死某一套工具 |
| 本机可跑 | Electron / 按 README-运行 拉起 |

隐喻：你做的是 **自己家里的净水器**——水（资料）自己灌，滤芯规则（拒答）自己定，机器装在自己环境里。

## 7. 明确不做（自用 / 认知期）

- 不做「开箱即有百科」的通识机器人  
- 不默认用 NAS 给外人当公网生产环境  
- 不先上自己的付费云托管  
- 不把「先卖出去」排在「自己能用、能涨认知」前面  
- 不接灰产 / 侵权资料入库  

## 8. 成功长什么样（自用）

- 至少喂进 1～2 个真实领域的资料，并日常会打开问  
- 归类结果自己看得懂、能搜到  
- 库内题能答、库外题会拒  
- Compose / 文档可复现；备份过至少一次  
- **以后**若要副业，再单独立项（见 [08](./08-扩展-对外交付.md)、[10](./10-扩展-市场与闸门.md)）
