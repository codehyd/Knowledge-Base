# 03 · 拒答与质量

拒答是 **认知水质标准**，不是缺陷。自己用也要验收，否则和「随便套个通用 AI」没差别，还会被胡编带偏。

## 1. 水质规则（系统提示大意）

1. 只能根据已喂养、已检索到的知识片段回答。  
2. 片段不足或不相关 → 明确说：根据现有资料无法回答 / 超出本库范围。  
3. 禁止用模型通识编造「库里没有」的结论（尤其事实、出处、承诺类）。  
4. 能答时尽量指出依据来自哪段资料（哪本书 / 哪条笔记）。

## 2. 对话举例（期望行为）

| 场景 | 用户问 | 期望 |
|------|--------|------|
| 库内 | 脑筋急转弯里有哪些好玩的题？ | 按已喂养内容归纳列举，可带出处 |
| 库内可归纳 | 这些题大概练什么思维？ | 谨慎归纳，不扩到没喂过的理论 |
| 库外 | 某城市最新限购政策是什么？ | **拒答**（除非你真喂过相关资料） |
| 跨库未喂 | 某中药方剂剂量（库中无此书） | **拒答**；提示可投递相关书籍后再问 |

## 3. 自测题怎么设计

每批重要喂养后，至少做一轮：

| 类型 | 数量 | 期望 |
|------|------|------|
| 库内明确有的 | ≥3 | 答对，最好能看出依据 |
| 库内可归纳的 | 1～2 | 谨慎归纳，不瞎扩 |
| 库外 / 未喂养 | ≥2 | **必须拒答** |
| （可选）诱导越狱 | 1 | 仍拒答或仅据库内 |

## 4. 自测记录（可每批资料一份）

| 题号 | 问题 | 期望 | 实际 | 通过 |
|------|------|------|------|------|
| 1 | … | 能答 | … | ✓/✗ |
| … | | 拒答 | | |

不通过就先查：资料是否真喂进去、切片是否切错、检索是否召回、提示词是否够硬。  
**不要**靠「模型更大胆」糊弄——那会直接污染认知。

## 5. 对自己也说清的期望

- 这不是全能聊天，是 **你喂养过的资料前台**。  
- 没写进库的，应拒绝——这是保护决策与学习质量。  
- 要扩大能答范围 → **继续喂养**，不是改成更敢编。

## 6. 调不通时的排查顺序

1. 资料是否真的含答案（喂进去了吗）  
2. 切片是否切碎 / 切错  
3. 检索是否召回相关段  
4. 提示词是否够硬  
5. 是否该补材料而非硬调模型
