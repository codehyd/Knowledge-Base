# 04 · 工具与运行

## 1. 原则

1. **本机 Electron + SQLite = 日常使用**（个人版默认，无 Docker）。  
2. **云托管 / NAS Compose 以后再说**（历史文件在 `archive/docker/`）。  
3. **Token / API Key 自己出**（设置页或 `.env`）；包里不塞真实密钥。

## 2. 用户要配什么才能对话

| 配置项 | 说明 |
|--------|------|
| 模型服务商 | 如 DeepSeek、其它 OpenAI 兼容接口 |
| API Key | 用户自己申请、自己保管 |
| （可选）Base URL / 模型名 | 按服务商文档填写 |

未配置 Key → 可浏览空库 / 上传入口，但 **入库 AI 处理与对话应不可用或明确提示去配置**。

## 3. 各放在哪

| 位置 | 用途 |
|------|------|
| 本机 Electron / Vite | 日常喂养、问答、自测 |
| `data/kongku.db` + uploads | 本地库与附件 |
| 备份目录 | 导出 ZIP（见 [05](./05-导出与备份.md)） |

## 4. 推荐启动

见仓库根 [README-运行.md](../../README-运行.md)：

```powershell
cd apps\desktop
npm run dev
```

首次可 `copy .env.example .env`，在设置页填 Key。

## 5. API Key 规范

- 仓库与备份 **不含** 真实 Key  
- 运行说明写清填哪几个变量（如 `LLM_API_KEY`、`LLM_BASE_URL`）  
- 试验用 Key 测完可轮换  

## 6. 备份习惯

日常备份优先拷贝 / 导出 `data/`（含 `kongku.db` 与 uploads），细节见 [05](./05-导出与备份.md)。
