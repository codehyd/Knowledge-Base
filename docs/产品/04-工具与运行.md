# 04 · 工具与运行

## 1. 原则

1. **自家 NAS + Docker = 日常使用与试验车间**。  
2. **云托管以后再说**；不把 NAS 默认暴露成公网生产。  
3. **Token / API Key 自己出**（`.env` 里自己的 Key，如 DeepSeek）；包里不塞真实密钥。

## 2. 用户要配什么才能对话

| 配置项 | 说明 |
|--------|------|
| 模型服务商 | 如 DeepSeek、其它 OpenAI 兼容接口 |
| API Key | 用户自己申请、自己保管 |
| （可选）Base URL / 模型名 | 按服务商文档填写 |

未配置 Key → 可浏览空库 / 上传入口，但 **入库 AI 处理与对话应不可用或明确提示去配置**。

## 3. 各放在哪

| 位置 | 用途 |
|------|------|
| 你的 NAS Docker | 日常喂养、问答、试 Compose、录屏自测 |
| 交付 / 备份目录 | Compose、说明、导出 ZIP（见 [05](./05-导出与备份.md)） |
| 你的云主机 | 现阶段不租；以后单独立项 |

## 4. 推荐目录模板

```
xxx-knowledge-docker/
  docker-compose.yml
  .env.example
  README-运行.md
  （可选）dockerfiles/
  （可选）scripts/import.sh
  （可选）seed/ 说明如何挂数据卷（默认空库，无预置知识）
```

典型命令：安装 Docker → 复制 `.env.example` 为 `.env` 填 Key → `docker compose up -d` → 浏览器打开端口 → 喂养第一份资料。

## 5. API Key 规范

- 仓库与备份 **不含** 真实 Key  
- 运行说明写清填哪几个变量（如 `LLM_API_KEY`、`LLM_BASE_URL`）  
- 试验用 Key 测完可轮换  
- 支持用户在界面里配置（自用期至少 `.env` 能通）

## 6. NAS 内存（如 8G）

**够自己轻量跑。**

- 跑应用 + DB，对话 / Embedding 用在线 API（Key 自备）  
- 不要在 NAS 上本地跑大模型  
- 少开其它吃内存套件  

## 7. 外网与演示

| 需求 | 做法 |
|------|------|
| 自己用 | NAS 局域网即可 |
| 给人看效果 | 优先录屏 / 截图，或会议共享 |
| 公网 Demo | 非必须；若做注意安全，用完关掉 |

## 8. 成本红线（自用）

- 固定成本目标：≈ 已有 NAS 电费  
- 模型费用按自己喂养 + 对话用量控  
- 不把带宽 / NAS 默认给外人当生产  

## 9. 技术选型（薄壳时）

- 网页前后端，Compose 一键起  
- DB 可用 Postgres + pgvector 等同机容器  
- 模型：OpenAI 兼容协议，用户 Key 路由  
- 不上 K8s；云以后再说
