# 05 · 导出与备份

资料在自己手里才安心——**能导出是长期可用的底线**。空库软件可以重装；喂进去的认知丢了才真正麻烦。

## 1. 自用期怎么做

每批重要喂养整理后，做一次备份。前期可 **手工打包**，不必先写导出按钮。

## 2. 导出什么

| 要 | 不要 |
|----|------|
| 原文 / 原文件（电子书、笔记） | API Key |
| 整理后的条目、分类、摘要（若有） | 无必要的向量文件 |
| 简单说明：如何再导入 | 「只有当前工具能打开」的加密壳 |

向量可不备份：换环境用自己的 Key **重建索引** 即可。心里要有数。

## 3. 推荐打包形式

`知识库备份-主题名-日期.zip`：

```
README.txt          # 导出说明、库名、日期
原文或整理/
  01-脑筋急转弯-摘录.md
  02-分类与摘要.jsonl
  …
（若有）原件/
  脑筋急转弯.pdf
```

小库也可再给一份合并的 `全部内容.md` 方便阅读。

## 4. 提醒自己

> 备份请放在 NAS 以外再留一份（如另一块盘 / 网盘）。问答索引可重建；原文丢了才真正麻烦。

## 5. 若以后有分享页

访客试用链接：**默认不允许访客导出全文**，避免被薅库。
