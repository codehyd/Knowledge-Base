# 06 · 工具与成本

## 1. 原则

1. **自家 NAS + Docker = 研发与自测车间**（你的 8G 够轻量试验）。  
2. **副业商品 = Docker 构建与运行交付**，跑在客户环境。  
3. **云托管以后再说**；不把你家 NAS 当默认生产公网。  
4. **Token 客户出**（`.env` 里客户自己的 Key）。

## 2. 两边分别干什么

| 位置 | 用途 |
|------|------|
| 你的 NAS Docker | 开发前后端、试 Compose、灌测试数据、录屏演示 |
| 交付包 | 给客户构建/运行；可含预置知识 |
| 客户电脑 / 客户 NAS / 客户云 | **生产运行** |
| 你的云主机 | 现阶段不租；以后单独立项 |

## 3. 交付物清单（建议固定模板）

```
xxx-knowledge-docker/
  docker-compose.yml
  .env.example
  README-运行.md
  （可选）dockerfiles/
  （可选）scripts/import.sh
  （可选）seed/ 说明如何挂数据卷
```

客户侧典型命令（写进 README）：安装 Docker → 复制 `.env.example` 为 `.env` 填 Key → `docker compose up -d` → 浏览器打开端口。

## 4. API Key

- 交付包 **不含** 真实 Key  
- 运行说明写清填哪几个变量  
- 你代测用自己的 Key，测完轮换/删除  

## 5. NAS 8G 够不够？

**够你自己做打包试验。**  

- 跑应用 + DB，对话/Embedding 用在线 API  
- 不要在 NAS 上本地跑大模型  
- 少开其它吃内存套件  

详见此前「内存速查」：8G 起步档；生产并发在客户机器上，不压你家。

## 6. 外网与演示

| 需求 | 做法 |
|------|------|
| 自己开发 | NAS 局域网访问即可 |
| 给客户看效果 | 优先 **录屏/截图**；或约定会议远程看你局域网 |
| 公网 Demo | 非必须；若做，注意安全，且勿当「我替你托管」 |

客户买的是包能跑，不是你家公网域名。

## 7. 以后若做云（闸门后）

才考虑：你租轻量云代托管、或「托管套餐」。  
与当前「卖 Docker 运行」分开定价，避免范围蔓延。见 [09](./09-30天计划与升级闸门.md)。

## 8. 成本红线

- 固定成本目标：≈ NAS 电费（已有设备）  
- 利润覆盖你的打包与协助时间  
- 不承诺用你的带宽/NAS 给客户当生产  

## 9. 技术选型（自研薄壳时）

- 网页前后端，Compose 一键起  
- DB 可用 Postgres+pgvector 等同机容器  
- 不上 K8s；云以后再说
