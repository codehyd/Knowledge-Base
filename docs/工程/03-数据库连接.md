# 数据库连接说明（开发环境）

本文记录 **当前本机开发** 用的 Postgres 配置，方便用 DBeaver / DataGrip / psql / Navicat 查看数据。

> 密码仅用于本地开发默认值；勿提交真实生产密钥。完整 Compose 生产形态见 [01-Docker与项目骨架](./功能/01-Docker与项目骨架.md)。

## 1. 当前连接参数（你这台机器）

本项目开发库占用标准端口 **5432**（容器 `kongku-db-1`）。

| 项 | 值 |
|----|-----|
| 类型 | PostgreSQL 16（镜像 `postgres:16-alpine`） |
| 容器名 | `kongku-db-1` |
| Host | `127.0.0.1` |
| Port | **5432** |
| Database | `kongku` |
| User | `kongku` |
| Password | `kongku` |
| JDBC | `jdbc:postgresql://127.0.0.1:5432/kongku` |
| URI（通用） | `postgresql://kongku:kongku@127.0.0.1:5432/kongku` |
| 应用异步 URL | `postgresql+asyncpg://kongku:kongku@127.0.0.1:5432/kongku` |

对应文件：

- 根目录 `.env` → `DATABASE_URL` / `POSTGRES_*`
- `docker-compose.dev.yml` → 只起 `db`，端口 `5432:5432`
- 全量 `docker-compose.yml` → api 容器内连 `db:5432`

### 端口曾被占用时的说明

此前本机 `5432` 被另一项目容器 **`knowledge-base-postgres-1`**（也是 Postgres）占用，空库临时用过 `5433`。  
现已停止该容器，**空库改回 5432**。若以后还要跑 `knowledge-base` 那套，需给它的 Postgres 改成其它端口（如 5434），避免再抢 5432。

## 3. 命令行查看

```bash
# 容器是否在跑
docker ps --filter name=kongku-db

# 进入 psql
docker exec -it kongku-db-1 psql -U kongku -d kongku

# 常用
\dt                 -- 表列表
\d ai_settings      -- 看表结构
SELECT id, provider, base_url, chat_model, embed_model,
       CASE WHEN api_key = '' THEN false ELSE true END AS has_key
FROM ai_settings;
```

本机若已装 `psql`：

```bash
psql "postgresql://kongku:kongku@127.0.0.1:5432/kongku"
```

## 4. 启停数据库

```bash
# Windows PowerShell 注意项目名（中文目录需指定）
$env:COMPOSE_PROJECT_NAME='kongku'
$env:POSTGRES_IMAGE='postgres:16-alpine'
docker compose -f docker-compose.dev.yml up -d
docker compose -f docker-compose.dev.yml down          # 停容器，数据卷保留
docker compose -f docker-compose.dev.yml down -v     # 慎用：删掉开发数据卷
```

数据卷名：`kongku_pgdata_dev`（持久化，重启不丢）。

## 5. 当前主要表（随功能增加）

| 表 | 说明 |
|----|------|
| `ai_settings` | 全局模型配置（通常 id=1）：provider、base_url、api_key、chat/embed 模型 |
| `entries` | 知识条目（喂养入库后使用） |
| `categories` | 分类字典 |

查看 API Key 时请只查「是否已配置」，避免在日志/截图里泄露明文：

```sql
SELECT id, provider, left(api_key, 3) || '***' AS key_preview, chat_model, embed_model
FROM ai_settings;
```

## 6. 和「全量 Docker」的区别

| 模式 | 数据库地址（应用内） |
|------|----------------------|
| 开发（本机 API + compose 只起 db） | `127.0.0.1:5432` |
| 全量 compose（api 也在容器里） | 主机名 `db`，端口 `5432`（见 `DATABASE_URL=...@db:5432/kongku`） |

换模式时改 `.env` 的 `DATABASE_URL`，并重启 API。
