# 数据库说明（个人版 · 默认 SQLite）

个人使用默认 **SQLite**，不依赖 Docker / Postgres。

## 1. 默认本地库

| 项 | 值 |
|----|-----|
| 类型 | SQLite |
| 文件 | `{DATA_DIR}/kongku.db`（默认仓库下 `data/kongku.db`） |
| 首次启动 | API 自动 `create_all` + 默认 `ai_settings` 行 |
| 应用 URL | `sqlite+aiosqlite:///<绝对路径>/kongku.db` |

留空 `.env` 里的 `DATABASE_URL` 即可走默认 SQLite。Electron 开发/打包亦默认本地库。

设置页可查看「表结构就绪」状态；必要时也可手动点「初始化表结构」。

可用任意 SQLite 客户端打开 `kongku.db` 查看表（如 DB Browser for SQLite）。

## 2. 主要表

| 表 | 说明 |
|----|------|
| `ai_settings` | 全局模型配置（通常 id=1） |
| `entries` / `categories` / `entry_categories` | 知识条目与分类 |
| `entry_annotations` | 条目笔记批注 |
| `chunks` | 切片与向量文本 |
| `sources` | 喂养任务 |
| `chat_sessions` / `chat_messages` | 对话 |

## 3. 可选：自备 Postgres

流程：**测试连接 → 保存并切换 → 点击「初始化表结构」**。  
切换连接时不会自动建表；初始化由你确认后触发（`POST /api/settings/db/init`）。

连接覆盖保存在 `data/runtime-db.json`，切换不会自动迁移数据。

历史 Docker Compose（仅作归档）见 `archive/docker/`。

## 4. 模式对照

| 模式 | 说明 |
|------|------|
| 个人默认 | SQLite；启动自动初始化 |
| 高级自备 | Postgres；保存后手动初始化 |
| 归档 | `archive/docker/` 里的 compose，默认流程不再启动 |
