# 02 · 架构约定（MVC · 按功能分模块）

目标：**改一个功能，尽量只动一个目录**；后续维护、优化、替换实现时成本低。

## 1. 总原则

| 原则 | 说明 |
|------|------|
| 按功能分包 | 喂养、对话、设置各自一个 module/feature，不按「全项目一层 Controller」堆文件 |
| MVC 分层 | Model 存数据 · Service 写业务 · Router/Page 做入口；互不跨层乱调 |
| 主入口要瘦 | `main.py` / `App.tsx` 只做组装，不写业务 |
| 共享要克制 | `core/`、`shared/` 只放真正跨功能的东西（配置、DB、布局、api client） |

## 2. 后端（FastAPI）结构

```text
apps/api/app/
  main.py                 # 组装路由
  core/                   # 配置、数据库、公共依赖
  modules/
    health/               # 示例功能
      router.py           # C：HTTP 入口
      service.py          # 业务
      schemas.py          # 入参/出参（View Model）
    settings_ai/
      models.py           # M：表结构
      schemas.py
      service.py
      router.py
    knowledge/
      models.py           # 条目等；后续补 service/router
    overview/
      ...
```

### 分层职责

| 层 | 文件 | 可以依赖 | 禁止 |
|----|------|----------|------|
| Model | `models.py` | SQLAlchemy、Base | 调 HTTP、调其它 Service |
| Schema | `schemas.py` | Pydantic | 碰 DB |
| Service | `service.py` | Model、Schema、httpx、其它 Service | 直接碰 FastAPI Request |
| Router | `router.py` | Service、Schema、Depends | 写复杂业务、直接拼 SQL |

### 新增功能（后端）检查单

1. 建 `modules/<name>/`
2. 需要表 → `models.py`，并在 `core/database.py` 的 `init_db` 里 import 一次
3. 写 `schemas.py` → `service.py` → `router.py`
4. `main.py`：`include_router(...)`
5. 补对应 `docs/工程/功能/xx.md` 验收项

## 3. 前端（React）结构

```text
apps/web/src/
  app/App.tsx             # 路由组装
  shared/                 # api client、布局、全局样式
  features/
    home/
    feed/
    chat/
    knowledge/
    settings/
```

| 层 | 位置 | 说明 |
|----|------|------|
| View | `features/*/XxxPage.tsx` + css module | 页面展示与交互 |
| 数据访问 | `shared/api/client.ts`（可再拆 `features/*/api.ts`） | 只谈 HTTP |
| 状态 | 先本地 `useState`；复杂了再按 feature 加 store | 避免全局大 store |

### 新增功能（前端）检查单

1. `features/<name>/` 建 Page + module.css
2. `App.tsx` 加路由
3. 若需接口：在 `shared/api/client.ts` 或 feature 私有 `api.ts` 增加方法
4. 对照 `figma/` 与工程功能文档

## 4. 为何这样利于维护

- **换解析器**（如 PDF 库）：只改喂养/解析 module 的 service  
- **换向量库**：只改 knowledge/chat 检索 service  
- **改拒答话术 UI**：只动拒答 feature + settings 模块  
- **升级某一屏**：不牵动其它 feature 的样式与状态
