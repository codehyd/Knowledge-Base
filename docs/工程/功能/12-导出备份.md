# 功能 12 · 导出备份

> UI：[figma/10-导出备份](../../../figma/10-导出备份.png)  
> 产品：[05-导出与备份](../../产品/05-导出与备份.md)

## 1. 功能目标

一键打包用户知识资产：原件 + 条目/分类/摘要；**永远不导出 API Key**；向量默认可不导（可重建）。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 格式 | ZIP；可选额外 `全部内容.md` |
| 内容勾选 | originals / entries / summaries；vectors 默认关；api_key **禁用且后端忽略** |
| 命名 | `知识库备份-{label}-{YYYY-MM-DD}.zip` |
| 实现 | 后端临时目录组装 → `FileResponse`；或写入 `data/exports` 供下载 |
| 再导入 | MVP 可只导出；导入作下一迭代（文档说明「可手工拷回 uploads + SQL」即可） |

## 3. 包内结构

```text
README.txt
原件/
  ...
整理/
  entries.jsonl
  categories.json
  （可选）全部内容.md
```

`README.txt` 写清日期、条目数、如何用自己的 Key 重建索引。

## 4. 实现步骤

1. `POST /api/exports` `{ include: {...} }` → job id 或同步小库直接返回
2. `GET /api/exports/{id}/download`
3. 前端勾选对齐 figma；Key 选项灰掉
4. 历史列表：扫 `data/exports` 或表记录

## 5. 验收

- [ ] ZIP 内无 `.env`、无 Key 字段
- [ ] 原件与条目能对上
- [ ] 大库导出不超时（过慢则改异步 job）
- [ ] 文档提醒：NAS 外再留一份
