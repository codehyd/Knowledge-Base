# 功能 04 · 喂养 · 电子书

> UI：[figma/02-喂养上传](../../../figma/02-喂养上传.png)  
> 产品：[02-喂养与归类](../../产品/02-喂养与归类.md)

## 1. 功能目标

用户上传 PDF / EPUB / TXT，系统保存原件并创建解析任务（真正的切片/摘要在功能 07）。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 上传 | `POST multipart/form-data`；限制大小（如 50MB） |
| 存储 | `data/uploads/{source_id}/original.ext` |
| 元数据 | 表 `sources`：`type=ebook`，`filename`，`status=pending` |
| 解析库 | PDF→`pypdf`；EPUB→`ebooklib`+html2text；TXT→直接读 |
| 编码 | 文本尝试 utf-8 / gbk |

## 3. 实现步骤

1. 前端拖拽区「电子书」，展示格式说明
2. API：`POST /api/sources/upload` → 落盘 + 插入 `sources`
3. 返回 `source_id`，队列显示「等待解析」
4. 触发解析任务（调用功能 07 入口）；本功能保证 **原件可追溯**
5. 失败：`status=failed` + `error_message` 可读

## 4. 正文抽取注意

- PDF 扫描件无文字层 → 标记失败，提示「当前版暂不支持纯图片 PDF」（OCR 可后置）
- 按页或按标题切大段，细切片交给功能 07

## 5. 验收

- [ ] 上传 PDF/EPUB/TXT 成功出现在队列
- [ ] 原件在 `data/uploads` 可找到
- [ ] 非法扩展名被拒绝
- [ ] 超限文件有明确错误
