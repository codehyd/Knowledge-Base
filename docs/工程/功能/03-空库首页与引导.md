# 功能 03 · 空库首页与引导

> UI：[figma/01-空库首页](../../../figma/01-空库首页.png)、[07-首次引导](../../../figma/07-首次引导.png)  
> 产品：[01-知识库定位](../../产品/01-知识库定位.md)

## 1. 功能目标

默认空库心智：无知识时首页引导去「喂养」或「配 Key」；可选三步首次引导（配 Key → 理解空库 → 喂第一份）。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 空库判断 | `SELECT COUNT(*) FROM entries`（或 sources 成功数）== 0 |
| Key 状态 | 读 settings，侧栏底部显示「已配置 / 未配置」 |
| 引导 | `localStorage.onboarding_done` 或 DB 标记；可跳过 |
| 文案 | 强调视频可自动提取文案、只按库内答 |

## 3. 实现步骤

1. `GET /api/stats/overview` → `{ entries, sources, key_configured }`
2. 首页：空态大标题 + 双 CTA（开始喂养 / 配置 API Key）
3. 非空态：可改为「最近喂养 / 去对话」简要入口（勿做成复杂仪表盘）
4. 引导弹层或独立路由 `/onboarding`：三步；完成写标记
5. 样式对齐 figma 主色 `#2A6F6A`，侧栏四导航

## 4. 验收

- [ ] 新库首次打开是空态文案，不是假数据
- [ ] 未配 Key 时侧栏有提示
- [ ] 引导可跳过且不再强制弹出（除非清标记）
