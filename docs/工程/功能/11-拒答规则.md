# 功能 11 · 拒答规则

> UI：[figma/11-拒答规则](../../../figma/11-拒答规则.png)  
> 产品：[03-拒答与质量](../../产品/03-拒答与质量.md)

## 1. 功能目标

把「只按库内答」做成可配置策略：严格模式、必须引用、是否允许库内归纳、拒答话术；对话链路强制遵守。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 存储 | `refusal_settings`：布尔开关 + `refuse_template` 文本 |
| 双闸门 | **检索闸门**（无相关 chunk 直接拒，不调 LLM）+ **提示词闸门**（有 chunk 也禁止通识发挥） |
| 默认 | 全部严格开关默认 ON；话术默认见产品文档 |
| 预览 | 设置页右侧用固定样例 Q 模拟两种回复（可本地前端 mock + 一次真实 test） |

## 3. 实现步骤

1. CRUD：`GET/PUT /api/settings/refusal`
2. Chat 服务读取设置：
   - 无命中 → 直接返回 template，不调用 chat 模型（省 Token）
   - 有命中 → system 注入规则（禁止通识、须引用等）
3. 前端开关与话术编辑；「恢复默认」
4. 与自测题绑定：改开关后复测库外题仍拒

## 4. 默认话术示例

```text
根据现有资料无法回答——超出本库范围。
你可以投递相关材料后再问，或换一个库内问题。
```

## 5. 验收

- [ ] 关闭「严格」前后行为可感知差异（但仍不应鼓励胡编）
- [ ] 无命中时不产生 chat 计费（可看日志）
- [ ] 话术修改立即生效
- [ ] **禁止**用「提高温度/更敢说」当作调通手段
