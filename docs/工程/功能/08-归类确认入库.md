# 功能 08 · 归类确认入库

> UI：[figma/03-归类确认](../../../figma/03-归类确认.png)  
> 产品：[02-喂养与归类](../../产品/02-喂养与归类.md)

## 1. 功能目标

展示 AI 草稿（摘要、标签、条目列表），允许用户改标签/勾选/编辑标题，确认后 **正式写入** entries 与分类关联；向量切片一并生效供检索。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 读草稿 | `GET /api/sources/{id}/draft` |
| 确认 | `POST /api/sources/{id}/confirm` body：用户改过的 tags、selected entry ids、summary |
| 事务 | 单事务：写 categories、entries、entry_categories；chunks 标记 `committed=true` |
| 人工优先 | 用户标签覆盖 AI；禁止「确认后不可改」（改分类见功能 09） |

## 3. 实现步骤

1. 前端步骤条停在「归类确认」
2. 左：原文/章节预览；右：摘要框、标签编辑器（增删）、条目表（checkbox）
3. 确认后 `status=committed`，跳转知识浏览并高亮新条目
4. 「返回重新解析」：清 draft，重回功能 07

## 4. 验收

- [ ] 不点确认则对话检索不到该批内容（或仅 draft 不可检索）
- [ ] 可删掉 AI 乱标签、自建「益智」
- [ ] 取消勾选的条目不入库
- [ ] 确认后 source 不可重复污染式确认（除非「追加」明确设计）
