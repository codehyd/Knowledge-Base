# 功能 05 · 喂养 · 笔记

> UI：[figma/02-喂养上传](../../../figma/02-喂养上传.png)  
> 产品：[02-喂养与归类](../../产品/02-喂养与归类.md)

## 1. 功能目标

支持 Markdown / 纯文本文件上传，或文本框粘贴；进入同一套 `sources` 解析流水线。

## 2. 技术要点

| 项 | 做法 |
|----|------|
| 入口 | 文件上传 **或** `POST JSON { title, content }` |
| 格式 | 统一存为 `.md` 原件，便于导出 |
| 切分倾向 | 按标题 `#` / 空行分段，利于「一条一事」 |

## 3. 实现步骤

1. 前端「笔记」区：上传 + 粘贴 Tab
2. API：
   - `POST /api/sources/upload`（`type=note`）
   - `POST /api/sources/paste` → 写文件 + 建 source
3. 状态机与电子书相同：`pending → processing → await_confirm / failed`
4. 解析时保留 Markdown 结构（代码块可整块切，避免切断）

## 4. 验收

- [x] 粘贴一段笔记可入库并完成正文抽取（`status=ready`；确认归类见功能 07/08）
- [x] 原件以 `.md` 落在 `data/uploads/{id}/`（导出能力后续）
- [x] 空内容提交被拒绝
