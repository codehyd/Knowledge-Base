# 功能 06 · 喂养 · 视频与链接（自动提取文案）

> UI：[figma/02](../../../figma/02-喂养上传.png)、[08-解析进度](../../../figma/08-解析进度.png)  
> 产品：[02-喂养与归类](../../产品/02-喂养与归类.md)（**无需用户手贴文稿**）

## 1. 功能目标

用户只贴视频/网页 URL → 系统 **自动**拿到文案 → 再进入切片/归类。  
仅当自动失败时，才允许用户补贴文案重试。

## 2. 技术策略（顺序很重要）

```text
1. 解析 URL 类型（B站/YouTube/通用页）
2. 尝试拉取字幕/CC（yt-dlp --write-subs / --write-auto-subs）
3. 若无字幕 → 调用云端 ASR API（用户 Key 或单独 ASR Key）
4. 若仍失败 → status=need_transcript，UI 提示「可补贴文案」
5. 成功 → 文案落盘 transcript.txt，转入功能 07
```

**禁止**：一上来就要求用户贴文稿当主路径。

## 3. 技术要点

| 项 | 做法 |
|----|------|
| 下载工具 | 容器内装 `yt-dlp`；注意站点 Cookies/合规，仅处理用户有权使用的内容 |
| 字幕 | 优先人工字幕，其次自动字幕；转纯文本 |
| ASR | OpenAI Whisper API 或服务商语音接口；**不要**默认本地 large 模型 |
| 网页 | 非视频链接用 trafilatura 抽正文 |
| 进度 | `sources.progress` + 阶段枚举：`fetch_meta` / `extract_caption` / `asr` / … |

## 4. 实现步骤

1. API：`POST /api/sources/url` `{ "url": "..." }`
2. Worker：按上序执行；每步写日志到 `source_events` 或日志字段
3. 前端队列：状态「提取文案中…」、徽章「自动转写」
4. 失败 UI：展示原因 + 文本框「粘贴文案后重试」→ `POST /api/sources/{id}/transcript`
5. 成功：与电子书一样进入解析流水线

## 5. 合规与风险

- 仅自用、合法内容；文档中写明用户自担版权责任
- 部分站点可能风控：失败走补贴路径即可，勿死磕破解

## 6. 验收

- [ ] 有字幕的视频链接：**零手贴**出文案并进解析
- [ ] 无字幕时走 ASR 或友好失败
- [ ] UI 不再出现「必须先提供文稿」作为默认文案
- [ ] 进度页能看到「提取字幕/语音转写」步骤
