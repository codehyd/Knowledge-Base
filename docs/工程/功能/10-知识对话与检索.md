# 功能 10 · 知识对话与检索

> UI：[figma/04-知识对话](../../../figma/04-知识对话.png)  
> 产品：[03-拒答与质量](../../产品/03-拒答与质量.md)

## 1. 功能目标

基于已入库切片做检索增强生成（RAG）：库内问题有据作答并带来源；库外问题拒答（细则见功能 11）。

## 2. 技术流程

```text
用户问题
  →（可选）分类过滤
  → embed(question)
  → SQL 向量相似度 top-k（如 k=5～8）
  → 分数/条数不足？→ 走拒答（功能 11）
  → 足够？→ 拼 system + 片段 + 提问 → 用户 Key chat
  → 返回 answer + citations[]
```

## 3. 技术要点

| 项 | 做法 |
|----|------|
| 检索 | pgvector `ORDER BY embedding <=> :q LIMIT k` |
| 阈值 | cosine；可设最小相似度阈值（如 0.35，需实测调） |
| 提示词 | 明确：只能依据片段；无依据必须拒答；引用 chunk id |
| 流式 | MVP 可非流式；进阶 SSE |
| 范围 | query `category_id`；中医库空时该过滤下更易拒答（正确行为） |

## 4. 实现步骤

1. `POST /api/chat` `{ message, category_id?, strict? }`
2. 组装 citations：`{ entry_id, title, source, snippet, score }`
3. 前端气泡展示引用 chip；拒答用独立样式（见图）
4. 记录会话可选（MVP 可不持久，或只存最近 N 条）
