# 00 · 技术选型

本文敲定 **空库** 自用 MVP 的技术栈。原则对齐产品文档：NAS + Docker、用户自备 Key、不在 NAS 跑大模型、视频自动提取文案。

## 1. 总栈（定稿）

| 层 | 选型 | 备选 | 为何定这个 |
|----|------|------|------------|
| 部署 | **Docker Compose** | — | 产品要求一键起；NAS 友好 |
| 前端 | **React + Vite + TypeScript** | Next.js | SPA 足够；构建简单，静态可由 Nginx/后端托管 |
| 后端 | **Python FastAPI** | Node NestJS | 文档解析、ASR、向量生态更熟；RAG 示例多 |
| 任务队列 | **ARQ**（基于 Redis）或 MVP 用 **BackgroundTasks + 表状态** | Celery | 喂养/转写耗时长，需异步；MVP 可先表驱动轮询 |
| 数据库 | **PostgreSQL 16 + pgvector** | SQLite+本地向量 | 同机容器即可；向量检索一步到位 |
| 对象/文件 | **本地数据卷** `./data` | MinIO | 自用无必要上对象存储 |
| LLM | **OpenAI 兼容 HTTP**（DeepSeek 等） | — | 用户自备 Key；Base URL 可配 |
| Embedding | 同一兼容接口的 embedding 模型 | 本地小模型 | NAS 内存紧，优先在线 API |
| 电子书解析 | **pypdf** + **ebooklib** + 纯文本 | unstructured | 覆盖 PDF/EPUB/TXT，依赖轻 |
| 视频文案 | **优先字幕**（yt-dlp 拉字幕）→ 失败再 **ASR API**（如 Deepgram/OpenAI Whisper API） | 本地 Whisper | **不强制用户贴文稿**；本地 Whisper 吃内存，自用期不推荐 |
| 网页抓取 | **httpx** + **trafilatura**（可选） | — | 链接类补充正文 |
| 反向代理 | Compose 内 **Nginx** 或由后端挂静态 | Caddy | 单端口对外即可 |

## 2. 明确不做（选型边界）

| 不做 | 原因 |
|------|------|
| K8s / 微服务拆太碎 | 自用单机，Compose 够 |
| 在 NAS 本地跑 7B+ LLM | 内存与稳定性不划算 |
| 绑定单一云厂商 SDK | 用 OpenAI 兼容层，换 DeepSeek/其它只改 Base URL |
| 预置通识向量库 | 违反空库原则 |
| 默认公网暴露 | 局域网自用 |

## 3. 推荐仓库结构

```
kongku/                          # 或 Knowledge-Base 下 app/
├── docker-compose.yml
├── .env.example
├── README-运行.md
├── apps/
│   ├── web/                     # React + Vite
│   └── api/                     # FastAPI
├── data/                        # gitignore：上传原件、导出临时包
└── docs/                        # 本知识库文档（已有）
```

## 4. 核心依赖（API 侧示意）

```text
fastapi, uvicorn
sqlalchemy / sqlmodel, asyncpg, pgvector
httpx, pydantic-settings
pypdf, ebooklib, python-multipart
yt-dlp（字幕）, 可选 openai 官方 SDK（兼容 base_url）
redis + arq（若上队列）
```

前端示意：`react`, `react-router`, `zustand` 或上下文, `tanstack-query`。

## 5. 数据对象（最小表）

| 表 | 作用 |
|----|------|
| `ai_settings` | Base URL、Key（加密或仅存本机）、模型名 |
| `sources` | 一次喂养任务：文件/URL、状态、进度 |
| `entries` | 知识条目（标题、摘要、所属 source） |
| `categories` / `entry_categories` | 多对多分类 |
| `chunks` | 切片正文 + embedding 向量 |
| `settings_refusal` | 拒答开关与话术 |
| `export_jobs` | 可选：导出任务记录 |

## 6. 与产品能力的映射

| 产品能力 | 技术落点 |
|----------|----------|
| 空库 | 无 seed 知识；仅空表 + 拒答默认配置 |
| 自备 Key | `ai_settings` + OpenAI 兼容客户端 |
| 电子书/笔记喂养 | 解析器 → 切片 → embed → 摘要分类 LLM |
| 视频自动文案 | yt-dlp 字幕 → 否则 ASR API |
| 只按库内答 | 检索 top-k chunks；不足则拒答提示词 |
| 可导出 | 打包 `data/` 原文 + entries JSON/MD，排除 Key |
