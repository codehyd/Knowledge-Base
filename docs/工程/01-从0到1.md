# 01 · 从 0 到 1

> **个人版现路径**：不再依赖 Docker。请直接按 [README-运行.md](../../README-运行.md)（Electron + SQLite）。  
> 下文保留早期「Compose + Postgres」建仓记录，便于对照 `archive/docker/`。

目标（历史）：从空仓库到 **浏览器能打开空库页 + API 健康检查通过 + 数据库可连**。  
不在本步实现喂养/对话；那些见 [功能/](./功能/)。

## 0. 前置

- 本机或 NAS 已装 Docker / Docker Compose
- 有一个可用的 DeepSeek（或其它兼容）API Key（先放 `.env`，界面配置见功能 02）
- 读过 [00-技术选型](./00-技术选型.md)

## 1. 初始化仓库

```bash
mkdir -p apps/web apps/api data
touch docker-compose.yml .env.example .gitignore README-运行.md
```

`.gitignore` 至少包含：

```gitignore
.env
data/
**/node_modules/
**/__pycache__/
.venv/
dist/
```

## 2. 写 `.env.example`

```env
# 运行期可被「设置页」覆盖；此处仅作首次启动默认
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_API_KEY=
LLM_CHAT_MODEL=deepseek-chat
LLM_EMBED_MODEL=deepseek-chat
# 若服务商有独立 embedding，改成对应模型名

POSTGRES_USER=kongku
POSTGRES_PASSWORD=kongku
POSTGRES_DB=kongku
DATABASE_URL=postgresql+asyncpg://kongku:kongku@db:5432/kongku
```

复制为 `.env` 并填入自己的 `LLM_API_KEY`（勿提交）。

## 3. `docker-compose.yml` 最小三件套

| 服务 | 镜像/构建 | 端口 |
|------|-----------|------|
| `db` | `pgvector/pgvector:pg16` | 5432（可仅内部） |
| `api` | build `apps/api` | 8000 |
| `web` | build `apps/web` 或 nginx 托管 dist | 5173/80 |

MVP 也可先本机 `uvicorn` + `vite`，Compose 只起 `db`；但 **目标形态仍是三服务 Compose**，与产品 [04](../产品/04-工具与运行.md) 一致。

## 4. API 最小代码

`apps/api`：

1. FastAPI `GET /health` → `{ "ok": true }`
2. 读取 Settings（pydantic-settings）
3. 启动时检查 DB 可连；执行建表或 Alembic 空迁移
4. 启用 CORS，指向 web 源
5. 启用 **Knife4j**：`/doc.html`（见 [功能/00-Knife4j接口文档](./功能/00-Knife4j接口文档.md)）

验收：

```bash
curl http://localhost:8000/health
# {"ok":true}
# 浏览器打开 http://localhost:8000/doc.html
```

## 5. Web 最小代码

`apps/web`：

1. Vite + React + TS
2. 布局壳：侧栏四项（喂养 / 对话 / 知识 / 设置），主色 `#2A6F6A`（见 figma）
3. 首页占位文案：「默认没有知识，需要你来喂养」
4. 请求 `GET /health` 显示「API 正常 / 异常」

对照：[figma/01-空库首页](../../figma/01-空库首页.png)

## 6. 跑通命令（写进 README-运行.md）

```bash
cp .env.example .env   # 填 Key
docker compose up -d --build
# 浏览器打开 http://localhost:<web端口>
```

- [ ] `README-运行.md` 按上面三步能复现
