# 04 · 交付流水线

## 1. 总流程

```
询盘（确认有运行环境）→ 收资料与验收题 → 你在 NAS 上开发/组装
  → 打出 Docker 交付包 → 客户侧构建运行（你可协助）
  → 验收题 → 交导出 ZIP 与文档 → 复盘
```

## 2. 分步

### Step 1 · 过滤

确认：垂直、套餐、客户是否有 Docker 环境；是否要加价代部署到客户机器。

### Step 2 · 收件

资料、必答题、必拒答题、客户 API Key 配置方式（只教他填 env，尽量不经手明文长期存放）。

### Step 3 · 你家 NAS 上组装

- 前后端/开源组件用 Docker 跑通  
- 导入垂直知识、配拒答  
- 自测验收题  

### Step 4 · 打交付物

至少包含：

| 内容 | 说明 |
|------|------|
| `docker-compose.yml`（及必要 Dockerfile） | 一键或少步启动 |
| `.env.example` | Key、端口等；无真实密钥 |
| `README-运行.md` | 环境要求、构建、启动、更新、常见问题 |
| （标准档）预置数据卷说明或导入脚本 | 知识如何进库 |
| 验收题清单 | 客户跑通后对照 |

可用压缩包或私有网盘发（注意别把你家生产密钥打进去）。

### Step 5 · 客户侧运行

- 文档指导客户 `compose up`  
- 或加价：你连客户机器远程装一次  
- **跑在客户资源上**

### Step 6 · 验收与备份

按 [05](./05-拒答与验收.md) 通过后，给 [07](./07-导出与信任.md) 的 ZIP。

### Step 7 · 复盘

耗时、Compose 哪步客户最卡 → 下周只改文档/脚本，不先上云托管。

## 3. 耗时目标（熟练后）

| 套餐 | 目标 |
|------|------|
| 体验（模板包） | 1～2h |
| 标准（含整理+跑通协助） | 4～8h |
| 加价代部署到客户云 | 另计 1～3h |

## 4. 交付话术示例

> 已按你的资料打好 Docker 知识库包。请按《运行说明》在你的环境启动；API Key 写入 `.env`。验收题见附件。另附资料备份 ZIP。我方不提供长期云托管；若需帮你装到你自己的服务器，可按加价项安排。
