# 01 · 副业定位

## 1. 你现在在做什么

| 角色 | 内容 |
|------|------|
| **副业卖什么** | **Docker 构建与运行交付**：一套可 `docker compose up` 的知识库（网页前后端等）+ 运行文档 +（可选）已整理好的垂直资料与拒答配置 |
| **你自己做什么** | 在家用 **NAS + Docker** 自研/试验打包；打磨镜像、Compose、说明 |
| **云端** | **以后再说**；当前不把「我来 24h 托管」当主卖点 |

客户付钱买的是：**能在他自己环境跑起来的结果**，不是扣子平替，也不是长期挂在你家 NAS 上的账号。

## 2. 三块边界（务必分清）

```
自家 NAS Docker     →  研发、自测、录演示
        ↓
交付物：镜像/Compose/文档/（可选）预置知识
        ↓
客户机器 / 客户云   →  客户自己构建与运行（你可远程协助一次）
```

| | 做 | 不做（现阶段） |
|--|----|----------------|
| 你家 NAS | 开发试验、自己验证 | 默认给所有客户当生产主机 |
| 交付 | 包 + 文档 + 教运行 | 承诺无限期免费代运维 |
| 云 | 客户若有云，帮他部署到他的云（可加价） | 你先租云当主业务成本 |

## 3. 和扣子等平台怎么相处

| | 扣子 / 云版工具 | 你的副业 |
|--|-----------------|----------|
| 形态 | 线上点点就能用 | **交 Docker，客户自托管** |
| 数据 | 多在平台云上 | **资料在客户自己机器** |
| 你的活 | 拼不过产品功能 | 垂直整理、拒答调好、Compose 打好、会教运行 |
| 话术 | 「强大 AI 平台」 | 「一套可运行的场景知识库 Docker，只按你的资料答」 |

仍可基于开源组件自研组装；对外卖的是 **你的打包与场景交付能力**。

## 4. 核心承诺

| 承诺 | 客户感知 |
|------|----------|
| Docker 能跑 | 按文档在本机/自己服务器拉起网页端 |
| 只答库内 + 会拒答 | 敢用于客服/教学 |
| 垂直资料可预置或代导入 | 不是交个空壳让他从零摸索 |
| 资料可导出 | 不绑死 |
| Key 自备 | 模型费用自己控 |

隐喻：你做的是 **净水器整机装箱说明书 +（可选）灌好的第一桶水**；机器装在客户家里，不装在你店里长期代管。

## 5. 分类规则（库内知识）

- 每条：**至少 1 个分类**，可以多个；能多则多，窄则一个，禁止硬凑。

## 6. 明确不做（副业期）

- 不对标扣子做通用云平台  
- 不默认用你家 NAS 给客户当公网生产环境  
- 不先上你自己的付费云托管（以后再说）  
- 不承诺通识万能答、不接灰产/侵权库  
- 不默认用你的 Token 给客户敞着用  

## 7. 成功长什么样

- 有可复用的 Compose/镜像模板  
- 闲鱼能卖「场景 Docker 包 + 协助运行」  
- 单笔：交包 → 客户侧跑通验收题 → 收尾  
- 以后若要做云托管，再单独立项（见 [09](./09-30天计划与升级闸门.md)）
