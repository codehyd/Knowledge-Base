# 11 · 附录：未来产品蓝图（非当前 KPI）

> 仅在通过 [09](./09-30天计划与升级闸门.md) 升级闸门后参阅。副业期不要按本章排期。

## 1. 方向（过闸门后）

在「Docker 包售卖」稳定后，才可选：

- 你方轻量云 **托管套餐**（另定价）  
- 或薄产品官网 + Freemium  

当前附录不指导开工顺序。

## 2. 能力分层（将来）

| 层 | 内容 |
|----|------|
| 入库 | 文本/文件为主；URL/视频以文稿为辅 |
| 净化 | 分类 1..N、摘要、切分、向量化 |
| 出水 | 检索问答、拒答、引用 |
| 信任 | 导出 ZIP、自备 Key |
| 增长 | 样板 Demo、分享访客页 |

## 3. 技术轮廓（将来选型参考）

- 前端：Web / H5；小程序后置  
- 后端：任选熟悉的 Node 或 Python  
- 数据：Postgres + pgvector（或等价）起步  
- 模型：OpenAI 兼容，用户 Key 路由  
- 部署：以 **Compose 交付客户自托管** 为主；你方云托管仅闸门后另议；不上 K8s
- 队列：入库异步（有量再上）

## 4. 数据对象（将来）

用户、AI 配置、知识库、条目、分类关联、切片、导出任务、（可选）会话与邀请码。

## 5. 与副业的关系

副业订单 = 需求探针与现金流。  
产品只解决「重复劳动与标准化交付」，不改变「不硬刚扣子」的战略。
